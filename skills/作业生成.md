# 作业生成规范

## 核心理念

**目标：巩固所学知识，通过练习让中小学生掌握编程基础**

1. **循序渐进**：3道简单题打基础，1道中等题提升
2. **优先洛谷**：优先使用洛谷 (luogu.com.cn) 在线judge的题目
3. **其次一本通**：其次使用《信奥一本通》配套题目
4. **都没有就出选择**：找不到合适题就出选择题
5. **答案要完整**：提供参考答案代码

## 二、题目搜索流程

### 题目来源优先级

1. **洛谷 (luogu.com.cn)**：在线评测题库，普及/入门难度
2. **信奥一本通**：配套教材题目
3. **都没有**：出选择题

### 搜索方法

#### 第一步：洛谷搜索

1. 访问洛谷，搜索与本课相关的关键词
2. 优先选择：
   - **难度标签**：普及-/普及/普及+
   - **通过率**：>50%（太难的不适合）
   - **题目类型**：涉及本课知识点的
3. 如果找到合适题目，直接使用

#### 第二步：信奥一本通

如果洛谷没有合适题目，搜索《信奥一本通》中的配套题目：
- 章节练习题
- 章节测试题
- 难度适中的题目

#### 第三步：选择题

如果都没有找到合适的题，转到选择题部分

## 三、选择题设计

### 适用场景

- 概念理解类课程
- 没有找到合适洛谷题
- 语法基础课程

### 出题原则

1. **覆盖本课重点**：每课核心知识点都要考
2. **选项要合理**：错误选项要像真实错误
3. **有区分度**：A/B/C/D都要看起来可能对
4. **4-6道选择题**为宜

### 常见错误选项

| 知识点 | 常见错误选项 |
|--------|------------|
| 循环次数 | 少算一次、多算一次、算成索引 |
| 变量作用域 | 以为循环外还能用 |
| 条件判断 | 边界值搞错 |
| 运算符优先级 | 没加括号导致顺序错 |

## 四、编程题设计

### 难度分布

| 难度 | 数量 | 特征 |
|------|------|------|
| 简单 | 3道 | 直接套用公式、照着例题改 |
| 中等 | 1道 | 需要简单变形或组合 |

### 简单题特征

- 题目直接说明要做什么
- 直接套用本课学的命令
- 不需要拆解复杂问题
- 10行以内能完成

### 中等题特征

- 需要用到之前学的内容
- 需要简单判断或循环
- 可能要处理边界情况
- 需要稍微想一想

### 题目描述规范

```markdown
**题目1：** 输出1到N的所有整数

**描述：** 输入一个整数N，输出所有从1到N的整数，每个数后面跟一个空格。

**示例输入：**
5

**示例输出：**
1 2 3 4 5

**提示：** 用for循环
```

## 五、语言规范

### 词汇对照表

| 应该用的词 | 避免用的词 |
|-----------|-----------|
| 命令 | 函数（除非解释清楚） |
| 显示、打印 | 输出 |
| 记住、存放 | 存储、赋值 |
| 读入、获取 | 输入 |
| 循环、重复 | 遍历 |
| 判断、如果 | 条件 |

### 代码注释规范

```cpp
// 注释解释"为什么"而不是"是什么"
// ✅ 好：让计数器从1数到10
for (int i = 1; i <= 10; i++)

// ❌ 差：循环10次
```

## 六、难度参考（以洛谷为基准）

### 简单题（洛谷普及-~普及）

- 直接输出
- 简单计算（加减乘除）
- 单层循环
- 简单条件判断

### 中等题（洛谷普及/普及+）

- 嵌套循环
- 需要积累结果（求和、计数）
- 简单算法思维（枚举、模拟）
- 组合本课多个知识点
