# 教案生成规范

## 核心理念

**目标：帮助零基础中小学生理解编程**

1. **通俗易懂**：用生活例子解释每个概念
2. **循序渐进**：从简单到复杂
3. **动手实践**：每节课都有练习
4. **问题拆解**：大问题拆成小问题

## 二、语言规范

### 词汇对照表

| 应该用的词 | 避免用的词 |
|-----------|-----------|
| 命令、指令 | 函数（除非解释清楚） |
| 显示出来、打印 | 输出、打印 |
| 记住、存放 | 存储、赋值 |
| 拆解问题 | 拆分问题 |
| 实际上在做什么 | 本质是什么 |
| 部件、组件 | 模块 |
| 一件什么事 | 什么操作 |
| 大盒子、小盒子 | 数组 |
| 卡片、便利贴 | 变量 |

### 常用生活类比库

| 概念 | 生活类比 |
|------|---------|
| 程序/代码 | 食谱、说明书 |
| 计算机处理 | 餐厅做菜（接单→做菜→上菜） |
| 命令/print | 遥控器按钮 |
| 变量 | 储物柜、便利贴 |
| 循环 | 重复做广播体操 |
| 判断/分支 | 遇到岔路口选择 |
| 函数 | 微波炉（放进去→设置→出来） |
| 递归 | 俄罗斯套娃 |
| 数组 | 一排编好号的储物柜 |
| 指针 | 取件码/钥匙 |
| 输入 | 点菜 |
| 输出 | 上菜 |

## 三、问题拆解流程

### 核心方法

```
遇到问题
  → 拆解问题（拆成小问题）
    → 再拆解
      → 直到每个小问题都能用代码解决
```

### 拆解原则

- 拆到"**一句话能说清楚**"为止
- 拆到"**一个命令能解决**"为止
- 拆到"**不需要再拆**"为止

### 拆解示例

**问题：让计算机显示 "Hello"**

```
1. 显示在哪里？→ 屏幕
2. 怎么让屏幕显示？→ 用一个"命令"
3. 什么命令？→ print() = "显示出来"

→ print("Hello")
```

**问题：计算 1+2+3+...+100**

```
1. 这是在做什么？→ 重复做加法
2. 重复多少次？→ 100次
3. 怎么让计算机重复？→ 用循环

→ 用 for 循环
```

## 四、概念解释方法

### 每个概念必须包含

1. **一句话通俗解释**（用生活类比）
2. **代码示例**（最简单的例子）
3. **"它在做什么"描述**（用"做什么"而不是"是什么"）

### 解释模板

```markdown
### XXX 是什么？

**通俗解释**：（一句话）
> 用生活中的例子帮助理解

**生活类比**：
> 就像______一样

**代码示例**：
```cpp
// 最简单的例子
```

**它在做一件什么事**：
> 描述代码的执行过程，用"做什么"而不是"是什么"
```

## 五、代码注释规范

### 注释应该解释"为什么"，而不是"是什么"

```cpp
// ❌ 不好：声明一个整数变量
// ✅ 好：用来存储学生年龄
int age;

// ❌ 不好：如果等于零
// ✅ 好：没有输入时（用户没给数字）
if (count == 0)

// ❌ 不好：循环10次
// ✅ 好：让计数器从1数到10（就像点名一样）
for (int i = 1; i <= 10; i++)
```

## 六、常见错误预防

### 每节课要包含

1. **学生常犯的错误**
2. **为什么会犯错**
3. **正确的写法**

### 示例

```markdown
## 常见错误

### ❌ 错误：忘记分号
```cpp
int a = 5  // 少了分号！
```
**为什么会错**：新手容易忽略分号
**正确写法**：`int a = 5;`

### ❌ 错误：数组下标越界
```cpp
int a[5];
a[5] = 10;  // 错误！最大是a[4]
```
**为什么会错**：没搞清楚从哪里开始数
**正确写法**：`a[4] = 10;`（从0开始数，5个元素是0~4）
```

## 七、练习题设计

### 分层设计

**基础题**：直接套用示例
**提高题**：需要简单变形
**挑战题**：需要综合运用

### 示例

```markdown
## 练习题

### 基础题
1. 写一个程序，显示 "Hello World"
2. 声明一个变量存储你的年龄

### 提高题
1. 写一个程序，交换两个变量的值
2. 计算 1+2+3+...+10 的和

### 挑战题
1. 写一个程序，判断年份是否是闰年
```

## 八、课堂互动设计

### 每节课应该有的互动

1. **导入时的提问**："同学们，你们知道计算机是怎么做加法的吗？"
2. **讲解时的互动**："这个应该放哪里？"
3. **练习后的检查**："做对的同学举手"
4. **总结时的回顾**："今天学了什么？谁能用话说一下？"

### 互动时间分配

| 环节 | 时间 | 互动类型 |
|------|------|---------|
| 导入 | 5分钟 | 提问、讨论 |
| 讲解 | 20分钟 | 提问、选择 |
| 练习 | 15分钟 | 动手操作 |
| 总结 | 5分钟 | 提问、复述 |
