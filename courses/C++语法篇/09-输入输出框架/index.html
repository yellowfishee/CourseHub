💡 关键问题

🔍 scanf 的秘密

scanf() 不仅能读数据，还能告诉我们"读成功了吗"

int result = scanf("%d", &x);

result = 1  // 成功读取了1个变量
result = 0  // 读取失败
result = EOF // 文件结束 (-1)

💡 提示2-19

Windows下输入完毕后按 Ctrl+Z，再按 Enter 结束。
Linux下按 Ctrl+D 结束。

✅ 标准写法

int x;
while(scanf("%d", &x) == 1) {
  // 处理 x
  printf("%d ", x);
}

✓ 正确

== 1 表示成功

✗ 错误

!= 1 或只写 scanf

🎯 测验

while(scanf("%d", &x) == 1) 什么时候退出循环？

A 当 x 等于 1 时
B 当用户按 Ctrl+C 时
C 当 scanf 返回值不等于 1 时
D 当 x 大于 1 时

🤖 算法竞赛的规则

  • • 有的比赛：程序从键盘读入，输出到屏幕（标准I/O）
  • • 有的比赛：程序从文件读入，输出到文件
  • • 有的比赛：禁止用重定向方式访问文件

⚠️ 提示2-22

不要弄错大小写，不要拼错文件名，不要使用绝对路径或相对路径！

🔗 方法1：重定向（简单）

freopen("data.in", "r", stdin);  // 读文件
freopen("data.out", "w", stdout); // 写文件

// 之后用 scanf 和 printf 就行了！

✓ 优点

代码不用改！原来怎么写scanf/printf，现在还是怎么写

🔗 方法2：fopen（灵活）

FILE *fin = fopen("data.in", "r");
FILE *fout = fopen("data.out", "w");

fscanf(fin, "%d", &x);  // 注意第一个参数是 fin
fprintf(fout, "%d\n", ans);

fclose(fin);
fclose(fout);

💡 提示2-24

如果不允许用重定向，就用 fopen 和 fscanf/fprintf

🔧 条件编译（推荐）

#define LOCAL
#include <stdio.h>

int main() {
#ifdef LOCAL
  freopen("data.in", "r", stdin);
  freopen("data.out", "w", stdout);
#endif
  // 程序代码...
}

💡 技巧

本地测试时定义 LOCAL，提交时删除 #define LOCAL 即可

🐛 常见Bug：变量未初始化

int x, min, max, s = 0;
while(scanf("%d", &x) == 1) {
  if(x < min) min = x;  // Bug! min未初始化
  if(x > max) max = x;
}

❌ 问题

min和max在未赋值前的值是不确定的！

⚠️ 提示2-20

变量在未赋值之前的值是不确定的。特别地，它不一定等于0。

✅ 解决方法

方法1：用"假想无穷大"初始化

const int INF = 1000000000;
int min = INF, max = -INF;

方法2：先读第一个数初始化

scanf("%d", &x);
int min = x, max = x;

🐛 多组数据变量重置

每组数据处理前，要重置变量！

while(scanf("%d", &n) == 1 && n) {
  int s = 0;  // 每组都要重置！
  int min = INF, max = -INF;
  for(int i = 0; i < n; i++) {
    scanf("%d", &x);
    s += x;
    if(x < min) min = x;
    if(x > max) max = x;
  }
}

⚠️ 提示2-26

在多数据题目中，一个常见的错误是计算完一组数据后某些变量没有重置
