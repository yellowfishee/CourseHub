🪆 俄罗斯套娃是什么？

大娃娃
中娃娃
小娃娃
小小娃娃

特点：每个娃娃里面都装着一个更小的自己

直到有一天...

最小的那个娃娃，里面是空的！

递归就像俄罗斯套娃！

🪆 俄罗斯套娃

打开一个大娃娃

→ 里面是一个中娃娃

→ 打开中娃娃

→ 里面是一个小娃娃

→ ...

→ 直到最小的娃娃（空的！）

💻 递归函数

调用一个大函数

→ 里面调用一个中函数

→ 中函数调用小函数

→ ...

→ 直到终止条件（不调用了！）

什么是递归？

递归 = 函数调用自己

void 打开娃娃() {
    打开娃娃(); // 调用自己！
}

⚠️ 等等！这会无限循环！

就像打开一个大娃娃，里面是中娃娃，打开中娃娃，里面是小娃娃...

如果没有"最小的那个是空的"，就会永远打开下去！

递归必须有两个条件！

✅ 终止条件

到最小娃娃时停止

"这个娃娃是空的，不用再打开了"

如果没有这个 = 无限循环 = 程序崩溃！

✅ 递归公式

大问题变小问题

"打开这个娃娃，里面还有娃娃"

每次都向终止条件靠近一步

用递归算阶乘！

什么是阶乘？

5! = 5 × 4 × 3 × 2 × 1 = 120

3! = 3 × 2 × 1 = 6

1! = 1

n! = n × (n-1)!

阶乘的递归思维 🔄

问：5! 是多少？

答：5! = 5 × 4!

那 4! 是多少？→ 4 × 3!

那 3! 是多少？→ 3 × 2!

那 2! 是多少？→ 2 × 1!

1! = 1（终止条件！）

所以：5! = 5 × 4 × 3 × 2 × 1 = 120

阶乘的递归代码

int factorial(int n) {
    if (n == 1) { // 终止条件
        return 1; // 最小娃娃是空的！
    }
    return n * factorial(n - 1); // 递归调用
}

调用 factorial(5) 时发生了什么？

→ factorial(5) = 5 × factorial(4)

→ factorial(4) = 4 × factorial(3)

→ factorial(3) = 3 × factorial(2)

→ factorial(2) = 2 × factorial(1)

→ factorial(1) = 1

factorial(5) 的调用过程

factorial(5)
= 5 × ?
→
factorial(4)
= 4 × ?
→
factorial(3)
= 3 × ?
→
factorial(2)
= 2 × ?
→
factorial(1)
= 1 ✓

然后往回算：

1 → 2×1=2 → 3×2=6 → 4×6=24 → 5×24=120

斐波那契数列

规律：每个数 = 前两个数之和

1, 1, 2, 3, 5, 8, 13, 21...

1 + 1 = 2, 1 + 2 = 3, 2 + 3 = 5, 3 + 5 = 8, 5 + 8 = 13...

Fib(n) = Fib(n-1) + Fib(n-2)

终止条件：Fib(1) = 1, Fib(2) = 1

斐波那契数列代码

int fib(int n) {
    if (n == 1 || n == 2) { // 终止条件
        return 1;
    }
    return fib(n - 1) + fib(n - 2); // 递归调用
}

fib(5) = fib(4) + fib(3)

= (fib(3)+fib(2)) + (fib(2)+fib(1))

= ((fib(2)+fib(1))+1) + ((1)+1)

= (1+1+1) + (1+1) = 5

🏛️ 汉诺塔问题

有三根柱子，最初所有盘子都在A柱子上

A柱
B柱
C柱

目标：把所有盘子移到C柱

汉诺塔规则 📋

✅ 可以做的
  • • 一次移动一个盘子
  • • 可以用B柱中转
  • • 大盘子不能放在小盘子上面
❌ 不能做的
  • • 一次移动多个盘子
  • • 大盘子放在小盘子上面
  • • 直接从A到C（不借助B）

3个盘子的解法

A
B
C

第1步：小盘 A→B

第2步：中盘 A→C

第3步：小盘 B→C

总共 3 步完成！

汉诺塔的递归思维 🧠

要把n个盘子从A移到C？

第1步：把上面的 n-1 个盘子从A借助C移到B

第2步：把最底下的大盘子从A直接移到C

第3步：把B上的 n-1 个盘子从B借助A移到C

hanoi(n) = hanoi(n-1) + 1 + hanoi(n-1)

= 2 × hanoi(n-1) + 1

汉诺塔递归代码

void hanoi(int n, char from, char to, char temp) {
    if (n == 1) { // 终止条件：只有1个盘子
        printf("%c → %c\n", from, to);
        return;
    }
    hanoi(n - 1, from, temp, to); // 第1步
    printf("%c → %c\n", from, to); // 第2步
    hanoi(n - 1, temp, to, from); // 第3步
}

调用 hanoi(3, 'A', 'C', 'B')

输出：A→B, A→C, B→C, A→B, C→A, B→C, A→B

汉诺塔的步数

盘子数最少步数公式
112⁰ = 1
232¹ + 2⁰ = 2+1 = 3
372² + 2¹ + 2⁰ = 4+2+1 = 7
4152³ + 2² + 2¹ + 2⁰ = 8+4+2+1 = 15

公式：hanoi(n) = 2ⁿ - 1

递归 vs 循环 🔄

🔁 循环

用 for 或 while

代码简洁

执行效率高

适合简单重复

🪆 递归

函数调用自己

代码简洁优雅

需要额外内存

适合分治问题

factorial 用循环也很简单：

int result = 1;
for (int i = 2; i <= n; i++) result *= i;

递归的调用栈 📚

调用栈就像一叠盘子：

fib(1)
↑
fib(2)
↑
fib(3)
↑
fib(4)
↑
fib(5)
← 后进先出 (LIFO)

fib(5) 需要调用 fib(4)，fib(4) 需要调用 fib(3)...

每一层调用都要等里面那一层算完才能返回

递归注意事项 ⚠️

❌ 没有终止条件

程序会无限递归，直到栈溢出崩溃！

⚠️ 终止条件写错了

比如 factorial(n) 写成 return n * factorial(n)

n 永远不会变小！

💡 每次递归要向终止条件靠近

参数要变小（n-1, n-2）或者满足终止条件

哪些是真正的递归？🤔

把是递归的拖到左边，是循环的拖到右边

用递归求和 1+2+3+...+n

递归公式：sum(n) = n + sum(n-1)

终止条件：sum(1) = 1

int sum(int n) {
    if (n == 1) return 1;
    return n + sum(n - 1);
}

计算：sum(5) = ?

sum(5) = 5 + sum(4)

= 5 + (4 + sum(3))

= 5 + 4 + (3 + sum(2))

= 5 + 4 + 3 + (2 + sum(1))

= 5 + 4 + 3 + 2 + 1
