💡 问题

服务员传菜有两种方式：

1. 把菜的副本端给顾客（顾客改了不影响厨房的菜）

2. 把菜的原件端给顾客（顾客改了厨房的菜也变了）

📄 什么是值传递？

值传递就像复印文件：原文件不变，拿到的是副本

📄

原件：a=10

main函数中的变量

→ 复印 →
📄📄

副本：a=10

swap函数中的变量

💡 关键点

修改副本不会影响原件！

📝 值传递示例

// 值传递的例子：修改不会影响原变量

void tryChange(int x) {
  // x是a的副本，不是a本身！
  x = x + 100;
  printf("函数里 x = %d\n", x);
}

int main() {
  int a = 10;
  tryChange(a);  // 把a的值传过去
  printf("函数外 a = %d\n", a);
  return 0;
}

函数里输出

x = 110

函数外输出

a = 10

💡 关键点

传递地址可以直接修改原变量！

📝 指针的声明和使用

// 指针声明：类型 * 指针变量名
int a = 10;  // 普通的整数变量
int *p;    // 声明一个指向int的指针
p = &a;     // p存储a的地址（&是取地址）

printf("a的值：%d\n", a);  // 10
printf("a的地址：%p\n", &a);
printf("p的值：%p\n", p);  // 和 &a 一样
printf("通过p访问a：%d\n", *p);// *是解引用，得到p指向的变量

&变量名

取地址运算符

*指针变量

解引用运算符

类型 *

🔄 经典问题：交换两个数

❌ 值传递（失败）

void swap(int a, int b) {
  int temp = a;
  a = b;
  b = temp;
}

// main中：
int x = 10, y = 20;
swap(x, y);
// 结果：x=10, y=20 ❌

✓ 指针传递（成功）

void swap(int *a, int *b) {
  int temp = *a;
  *a = *b;
  *b = temp;
}

// main中：
int x = 10, y = 20;
swap(&x, &y);
// 结果：x=20, y=10 ✓

🔍 为什么指针传递能成功？

内存示意图

main函数

x: 10
y: 20
↓ &x, &y

swap函数

*a: → x
*b: → y

💡 关键

指针 a 指向 x，指针 b 指向 y。
修改 *a 就是修改 x，修改 *b 就是修改 y！

📊 数组作为参数

数组名本身就是地址！传递给函数时不需要 &

void printArray(int arr[], int n) {
  // arr 是指针，指向数组首元素
  for(int i = 0; i < n; i++)
    printf("%d ", arr[i]);
}

int main() {
  int a[5] = {1, 2, 3, 4, 5};
  printArray(a, 5);  // a 就是地址，不需要 &a
  return 0;
}

💡 重要特性

数组作为参数时是地址传递，在函数内部修改数组会影响原数组！

🧠 内存布局图

值传递

栈内存：

┌─────────────────────┐
│ main: a = 10 │
├─────────────────────┤
│ swap: a = 10 (副本) │ ← 独立的变量
│ swap: b = 20 (副本) │
└─────────────────────┘

修改副本，不影响原件

指针传递

栈内存：

┌─────────────────────┐
│ main: x = 10 │
│ main: y = 20 │
├─────────────────────┤
│ swap: a → &x │ ← 指向x
│ swap: b → &y │ ← 指向y
└─────────────────────┘

通过地址直接操作原件

📋 本章总结

值传递

  • • 传递数据的副本
  • • 修改副本不影响原变量
  • • 函数内部修改无效
  • • 普通变量传递方式

指针传递

  • • 传递变量的地址
  • • 通过地址直接修改原变量
  • • 用 & 取地址，用 * 解引用
  • • 数组天然是地址传递

🎯 核心要点

1️⃣

&取地址运算符

&x 表示变量 x 的内存地址

2️⃣

*解引用运算符

*p 表示访问 p 指向的变量的值

3️⃣

swap(&x, &y) 的 & 不能省

&x 把 x 的地址传过去，函数才能直接修改 x

4️⃣

数组名本身就是地址

传递给函数时不需要加 &

⚠️ 常见错误

❌ 错误1：指针未初始化

int *p;
*p = 10;  // 错误！p是野指针，指向未知内存

❌ 错误2：忘记取地址 &

swap(x, y);  // 错误！应该是 swap(&x, &y)

❌ 错误3：解引用空指针

int *p = NULL;
*p = 10;  // 错误！不能解引用NULL指针
