💡 关键发现

嵌套循环就像流水线，时间是相乘的关系！

🧮 阶乘之和问题

计算 S = 1! + 2! + 3! + ... + n! 的末6位

for(int i = 1; i <= n; i++) {
  int factorial = 1;
  for(int j = 1; j <= i; j++)  // 嵌套循环！
    factorial *= j;
  S += factorial;
}

❌ n=100

输出: -961703

（溢出了！）

⏰ n=10^6

需要5小时！

（太慢了！）

⏱️ 如何测量时间？

C语言提供了一个计时神器：clock() 函数

#include <time.h>

// 在程序结束前调用
printf("Time used = %.2f\n",
  (double)clock() / CLOCKS_PER_SEC);

⚠️ 注意

要除以 CLOCKS_PER_SEC，因为不同系统单位不同！

n每翻倍，时间大约增加4倍！这就是O(n²)复杂度！

🔧 问题1：整数溢出

当 n=100 时，阶乘会变得巨大无比：

100! = 93,326,789,638,259,136,990,902,140...
远远超过 int 能表示的范围 (-21亿 ~ +21亿)

💡 提示2-16

要计算表达式的余数，可以在每步计算后取模，结果不变！

✅ 解决方案：每步取模

const int MOD = 1000000;

for(int i = 1; i <= n; i++) {
  int factorial = 1;
  for(int j = 1; j <= i; j++)
    factorial = (factorial * j) % MOD;  // 每次取模！
  S = (S + factorial) % MOD;
}

✓ n=100

正确输出！

✓ n=1000

也能算！

💡 为什么不变了？

因为 25! 末尾有 6个0，所以从第5项开始，后面的阶乘对末6位都没有影响！

🧮 为什么25!末尾有6个零？

一个数末尾的0来自因子 10 = 2 × 5

25! = 1×2×3×4×5×...×24×25

包含的5有: 5, 10, 15, 20, 25(有2个5)
共 6 个5！

所以 25! = ...0000000 (6个0)

💡 优化技巧

如果 n > 25，可以直接设 n = 25，速度快10000倍！

🎯 优化后的代码

const int MOD = 1000000;
int n, S = 0;
scanf("%d", &n);

if (n > 25) n = 25;  // 优化！

for(int i = 1; i <= n; i++) {
  int factorial = 1;
  for(int j = 1; j <= i; j++)
    factorial = (factorial * j) % MOD;
  S = (S + factorial) % MOD;
}
