循环的代价

程序运行时间怎么算？

⏰ 时间复杂度 📈 O(n²) ⚡ 优化技巧

循环需要"时间" ⏰

🏭

想象工厂流水线

1个工人做1个产品：1分钟
10个工人做10个产品：10分钟
100个工人做100个产品：100分钟

产品越多，需要的时间越多！

计算机也一样！💻

计算机很快，但循环太多也会变慢！

100

次循环 → 很快

10,000

次循环 → 还行

1,000,000

次循环 → 慢！

嵌套循环：更慢！🐌

📦📦

嵌套循环就像流水线

外层循环10次 × 内层循环10次 = 100次
就像10个工人，每个做10件产品！ 

for (int i = 1; i <= 10; i++) {       // 外层：执行10次
    for (int j = 1; j <= 10; j++) {   // 内层：每次执行10次
        // 做事
    }
}
// 总共：10 × 10 = 100 次

什么是时间复杂度？📊

时间复杂度就是：当数据变大，时间怎么变化？

简单循环

n 次

叫 O(n)

嵌套循环

n × n 次

叫 O(n²)

O 念作 "大O"，表示"大约是..."

测验：复杂度 🧠

下面代码的时间复杂度是多少？

for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= n; j++) {
        // 做事
    }
}
A O(n)
B O(n²)
C O(1)

O(n²) 有什么问题？⚠️

当 n 变大时，O(n²) 会变得非常慢！

n 的大小 O(n) 的次数 O(n²) 的次数
10 10 100
100 100 10,000
1000 1,000 1,000,000

💡 n=1000时，O(n²)比O(n)慢1000倍！

循环的另一个问题：数字太大！💥

💥

溢出问题

数字太大会装不下！ 

int factorial = 1;
for (int i = 1; i <= 20; i++) {
    factorial = factorial * i;  // 20! 已经超过 int 范围了！
}

💡 int 只能装约 -21亿 到 +21亿

解决办法：取模！🔄

取模就像"只保留末尾"：

12345

原始数字

345

对 1000 取模

factorial = (factorial * i) % 1000000;

✓ 取模后数字永远不超过 1000000，不会溢出！

测验：取模 🧠

123 对 10 取模，结果是多少？

A 12
B 3
C 123

提示：取模就是除法取余数！

填空练习：取模运算 ✏️

填入正确的取模符号：

factorial = factorial * i ? 1000000;

*

神奇的发现！✨

观察阶乘之和的结果：

n=10

37913

n=25

940313

n=100

940313

n=25、30、100... 结果都一样！

为什么结果一样？🔍

因为 25! 末尾有 6 个零！

阶乘的"末尾零"来自因子 10 = 2 × 5 

// 1到25里有多少个5？
// 5, 10, 15, 20, 25(有2个5) = 6个5！
// 所以 25! = xxx000000 (6个零)

从第6项开始，累加的末尾6位都是0！

优化技巧！⚡

如果 n > 25，可以直接设 n = 25！ 

if (n > 25) {
    n = 25;  // 优化！
}

优化前

n=100 → 很久

优化后

n=25 → 飞快！

测验：优化 🧠

为什么要设 n = 25？

A 因为 n > 25 时，25! 末尾已经有6个零，结果不变
B 因为 25 是一个特殊的数字
C 因为 25 是最快的计算

怎么测量程序时间？⏱️

C语言有一个计时命令叫 clock() 

#include <time.h>

// 程序结束时调用：
printf("时间 = %.2f 秒\n", 
    (double)clock() / CLOCKS_PER_SEC);

💡 CLOCKS_PER_SEC 是每秒的时钟数
