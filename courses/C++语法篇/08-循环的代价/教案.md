# 教案：08-循环的代价

## 教学目标

1. **理解循环的效率问题**: 通过阶乘求和的例子，让学生看到嵌套循环导致的"平方"级别时间复杂度
2. **掌握计时工具**: 学会使用 `clock()` 函数测量程序运行时间
3. **学会防溢出技巧**: 在每步计算后取模，避免整数溢出
4. **认识数学规律**: 25!以后末6位都是0

## 引入阶段 (5分钟)

### 生活类比：快递分拣

想象你是快递站的分拣员：
- 只有1个包裹时：1分钟分完
- 有10个包裹时：需要10分钟（每个都要检查）
- 有100个包裹时：需要100分钟吗？

**不是！** 如果你用流水线方式：
- 第1个人分拣第1个
- 第2个人分拣第2个...
这就像**嵌套循环**，时间是相乘的关系！

## 讲解阶段 (15分钟)

### 核心例题：阶乘之和

计算 S = 1! + 2! + 3! + ... + n! 的末6位

**错误代码演示**（故意有bug）:
```c
#include <stdio.h>
int main() {
    int n, S = 0;
    scanf("%d", &n);
    for(int i = 1; i <= n; i++) {
        int factorial = 1;
        for(int j = 1; j <= i; j++)  // 嵌套循环！
            factorial *= j;
        S += factorial;
    }
    printf("%d\n", S % 1000000);
    return 0;
}
```

**问题1**: 当 n=100 时，输出 -961703（错误！溢出）
**问题2**: 当 n=10^6 时，需要近5小时！（太慢！）

### 提示提取

- **提示2-15**: 在循环体开始处定义的变量，每次执行循环体时会重新声明并初始化
- **提示2-16**: 要计算只包含加法、减法和乘法的整数表达式除以正整数n的余数，可以在每步计算之后对n取余，结果不变
- **提示2-17**: 可以使用time.h和clock()函数获得程序运行时间
- **提示2-18**: 很多程序的运行时间与规模n存在着近似的简单关系

## 交互练习 (15分钟)

### 练习1: 计时实验

让学生修改代码，加上计时器：
```c
#include <time.h>
// ...
printf("Time used = %.2f\n", (double)clock() / CLOCKS_PER_SEC);
```

观察不同n值的运行时间，验证"时间与n²成正比"的规律。

### 练习2: 取模防溢出

把 `factorial *= j` 改成 `factorial = (factorial * j) % 1000000`

### 练习3: 找规律

测试不同的n值，观察从哪个n开始答案不变（答案是25，因为25!末尾有6个0）

## 总结 (5分钟)

### 本课要点

1. **嵌套循环的代价很大**: 时间复杂度是 O(n²)
2. **clock()计时**: 可以精确测量程序运行时间
3. **取模防溢出**: 每步取模不改变最终结果
4. **数学常识**: 25!末尾有6个0

### 课后思考

- 如果n=1000，如何优化这个程序？
- 为什么要用 `double` 除以 `CLOCKS_PER_SEC`？

## 教学方法

1. **先演示bug**: 让学生看到错误结果，激发学习兴趣
2. **逐步调试**: 用输出中间结果的方法找到问题
3. **动手实验**: 学生自己修改代码，观察时间变化
4. **归纳规律**: 从实验数据中发现"平方"关系
