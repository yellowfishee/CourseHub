📦 函数基础：什么是函数？

函数是一段可以重复使用的代码，它接受输入（参数），进行处理，然后返回结果。

📥

🏭

💡 生活类比

就像自动售货机：投入硬币（参数），机器进行处理，弹出商品（返回值）。函数就是程序的"自动售货机"！

📝 函数的定义语法

// 函数定义的基本格式
返回类型 函数名(参数类型 参数名1, 参数类型 参数名2, ...) {
  // 函数体：具体的处理步骤
  // 可以有多条语句
  return 结果值;  // 返回结果
}

示例：求最大值

int max(int a, int b) {
  if (a > b)
    return a;
  return b;
}

示例：打印问候语

void greet(char name[]) {
  printf("Hello, %s!\n", name);
}

📞 函数调用过程详解

当调用一个函数时，程序会执行以下步骤：

1️⃣

暂停当前程序

调用函数时，主函数的执行暂停

2️⃣

传递参数

将实参的值复制给形参（值传递）

3️⃣

执行函数体

按照代码顺序执行函数内的语句

4️⃣

返回结果

将返回值带给调用者，恢复主函数执行

📦 参数传递：值传递

值传递就像复印文件：原文件不变，函数拿到的是副本

void tryChange(int x) {
  x = x + 100;  // 修改的是副本！
  printf("函数内 x = %d\n", x);
}

int main() {
  int a = 10;
  tryChange(a);  // 传递a的值（副本）
  printf("函数外 a = %d\n", a);
  return 0;
}

函数内输出

x = 110

函数外输出

a = 10

💡 关键点

值传递时，函数内部修改的是参数的副本，不会影响外部的原始变量！

📍 参数传递：指针传递

指针传递就像给机器钥匙：可以通过钥匙直接操作原文件

void changeDirectly(int *p) {
  *p = *p + 100;  // 通过地址直接修改原变量！
  printf("函数内 *p = %d\n", *p);
}

int main() {
  int a = 10;
  changeDirectly(&a);  // 传递a的地址
  printf("函数外 a = %d\n", a);
  return 0;
}

函数内输出

*p = 110

函数外输出

a = 110

💡 关键点

指针传递时，函数可以直接修改外部变量！使用 & 获取地址，使用 * 访问值。

🔄 经典例题：交换两个数

❌ 值传递（错误）

void swap(int a, int b) {
  int temp = a;
  a = b;
  b = temp;
}

// main中调用
int x = 10, y = 20;
swap(x, y);
// 结果：x=10, y=20 （没变！）

✓ 指针传递（正确）

void swap(int *a, int *b) {
  int temp = *a;
  *a = *b;
  *b = temp;
}

// main中调用
int x = 10, y = 20;
swap(&x, &y);
// 结果：x=20, y=10 （交换成功！）

📊 数组作为参数

数组名本身就是地址，传递给函数时不需要 & 符号！

void printArray(int arr[], int n) {
  // arr 是一个指针，指向数组首元素
  for(int i = 0; i < n; i++)
    printf("%d ", arr[i]);
}

int main() {
  int a[5] = {1, 2, 3, 4, 5};
  printArray(a, 5);  // a 就是地址，不需要 &a
  return 0;
}

💡 重要特性

数组作为参数时是地址传递，在函数内部修改数组会影响外部的原始数组！

🏠 结构体：自定义数据类型

结构体就像一个"数据容器"，可以把相关的数据打包在一起

📋

学生信息

姓名："小明"

年龄：15

分数：95.5

// 定义结构体类型
struct Student {
  char name[50];  // 姓名（字符数组）
  int age;          // 年龄（整数）
  double score;      // 分数（浮点数）
};

🖊️ 结构体的使用

// 1. 声明结构体变量
struct Student s1, s2;

// 2. 给成员赋值（使用点号 . 访问）
strcpy(s1.name, "小明");
s1.age = 15;
s1.score = 95.5;

// 3. 访问成员并打印
printf("姓名：%s\n", s1.name);
printf("年龄：%d\n", s1.age);
printf("分数：%.1f\n", s1.score);

输出结果：

姓名：小明

年龄：15

分数：95.5

🔧 结构体与函数

结构体可以作为函数的参数和返回值使用

结构体作为参数

void printPoint(struct Point p) {

}

结构体作为返回值

struct Point createPoint(int x, int y) {
  struct Point p = {x, y};

}

💡 提示

结构体作为参数时是值传递（传递副本），如果结构体很大，复制会花费较多时间

💡 问题

如果没有控制，递归会无限进行下去！所以递归必须要有终止条件！

🎯 递归的两个必要要素

✅

终止条件

递归结束的条件

当满足这个条件时，函数不再调用自己，直接返回结果

if (n == 1) return 1;
🔁

递归公式

将大问题分解为小问题

函数调用自己来处理更小规模的问题

return n * factorial(n-1);

⚠️ 警告

没有终止条件的递归会无限调用，直到程序崩溃（栈溢出）！

📝 递归实例：阶乘

n! = n × (n-1) × (n-2) × ... × 2 × 1

// 阶乘的递归实现
int factorial(int n) {
  // 终止条件：1! = 1
  if (n == 1) return 1;
  
  // 递归公式：n! = n × (n-1)!
  return n * factorial(n - 1);
}

factorial(5) 的计算过程：

factorial(5) = 5 × factorial(4)

= 5 × 4 × factorial(3)

= 5 × 4 × 3 × factorial(2)

= 5 × 4 × 3 × 2 × factorial(1)

= 5 × 4 × 3 × 2 × 1 （终止条件返回）

= 120

🧠 递归调用栈

递归调用使用"后进先出"的栈来管理，每次调用都会在栈上创建一个新的"帧"

factorial(3) 的调用顺序：

第1层： factorial(3) 调用 factorial(2)
第2层： factorial(2) 调用 factorial(1)
第3层： factorial(1) 返回 1（终止）
返回1给第2层
第2层： 返回 2×1=2
第1层： 返回 3×2=6

调用栈的变化（从下往上）：

┌─────────────────┐
│ factorial(1) │ ← 最后调用，最先返回
│ 返回值: 1 │
├─────────────────┤
│ factorial(2) │
│ 返回值: 2 │
├─────────────────┤
│ factorial(3) │ ← 最先调用，最后返回
└─────────────────┘

📝 递归实例：斐波那契数列

斐波那契数列：1, 1, 2, 3, 5, 8, 13, 21, ...

规则：F(n) = F(n-1) + F(n-2)，前两个数都是1

// 斐波那契数列的递归实现
int fib(int n) {
  // 终止条件：前两个数都是1
  if (n <= 2) return 1;
  
  // 递归公式：F(n) = F(n-1) + F(n-2)
  return fib(n - 1) + fib(n - 2);
}

fib(5) 的计算过程：

fib(5)

= fib(4) + fib(3)

= (fib(3)+fib(2)) + (fib(2)+fib(1))

= (fib(2)+fib(1)+1) + (1+1)

= (1+1+1) + (1+1)

= 5

计算过程图示

fib(5)

├─ fib(4)

│ ├─ fib(3)

│ │ ├─ fib(2) = 1

│ │ └─ fib(1) = 1

│ └─ fib(2) = 1

└─ fib(3)

├─ fib(2) = 1

└─ fib(1) = 1

📝 递归实例：数字各位求和

计算一个整数各位数字之和

例如：12345 → 1+2+3+4+5 = 15

// 数字各位求和的递归实现
int digitSum(int n) {
  // 终止条件：一位数直接返回
  if (n < 10) return n;
  
  // 递归公式：各位和 = 最低位 + 剩余位的和
  return n % 10 + digitSum(n / 10);
}

digitSum(456) 的计算过程：

digitSum(456)

= 6 + digitSum(45)

= 6 + 5 + digitSum(4)

= 6 + 5 + 4 （终止条件返回）

= 15

📝 递归经典：汉诺塔

将n个盘子从A柱移动到C柱，规则：大盘子不能放在小盘子上

void hanoi(int n, char from, char to, char temp) {
  if (n == 0) return;  // 终止条件
  
  // 1. 将n-1个盘子从from移到temp（借助to）
  hanoi(n - 1, from, temp, to);
  
  // 2. 将最大的盘子从from移到to
  printf("%c -> %c\n", from, to);
  
  // 3. 将n-1个盘子从temp移到to（借助from）
  hanoi(n - 1, temp, to, from);
}

hanoi(3, 'A', 'C', 'B') 的输出：

A -> C

A -> B

C -> B

A -> C

B -> A

B -> C

A -> C

共移动 2³-1 = 7 次

汉诺塔移动公式

n个盘子需要移动 2ⁿ - 1 次

n=1: 1次

n=2: 3次

n=3: 7次

n=4: 15次

n=5: 31次

🔄 递归 vs 循环

递归的优点

  • ✓ 代码简洁优雅
  • ✓ 容易理解（对于适合递归的问题）
  • ✓ 天然适合"分解问题"的场景
  • ✓ 树结构、图结构的遍历

递归的缺点

  • ✗ 函数调用有开销
  • ✗ 占用栈空间（可能有栈溢出风险）
  • ✗ 重复计算（如斐波那契）
  • ✗ 调试可能较困难

💡 建议

能用循环解决的问题优先用循环，递归更适合"问题本身具有递归结构"的场景（如树遍历、分治算法）。

⚠️ 常见错误总结

❌ 错误1：忘记终止条件

// 错误：没有终止条件，会无限递归！
int badRecursive(int n) {
  return n + badRecursive(n - 1);
}

❌ 错误2：忘记返回类型

// 错误：声明返回int但没有return语句
int forgetReturn(int x) {
  printf("%d\n", x);
  // 错误：没有return语句！
}

❌ 错误3：结构体成员访问错误

// 错误：指针变量应该用 -> 而不是 .
struct Student *p;
p->name  // 正确
p.name   // 错误！
