# 19-注解与习题 - 教案

---

## 教学目标

1. 知识目标：全面掌握函数、结构体和递归的知识体系
2. 能力目标：能够正确分析递归调用过程，设计递归算法
3. 情感目标：培养严谨的编程思维和良好的编码习惯

## 教学重点

- 函数的值传递和指针传递的区别
- 结构体的定义和使用
- 递归的两个要素：终止条件和递归公式
- 递归调用栈的理解

## 教学难点

- 递归调用过程的深入理解
- 指针传递与值传递的区别
- 汉诺塔等经典递归问题

## 教学过程

### 导入（10分钟）

回顾函数、结构体、递归三章的知识框架。

### 新课讲解（35分钟）

**函数部分（10分钟）**
- 函数定义和调用的完整过程
- 值传递和指针传递的对比
- 数组作为参数的特殊性

**结构体部分（10分钟）**
- 结构体的定义和成员访问
- 结构体与函数的关系

**递归部分（15分钟）**
- 递归的两个必要要素
- 阶乘、斐波那契、数字求和等经典实例
- 递归调用栈的演示
- 递归与循环的比较

### 常见错误分析（10分钟）

展示学生容易犯的错误及其纠正方法。

### 课堂测验（5分钟）

检验学生对重要概念的理解。

## 提示与答案

### 提示

1. 递归必须有终止条件，否则会无限递归导致栈溢出
2. 值传递传递副本，指针传递传递地址
3. 结构体用 . 访问成员，指针用 ->
4. 数组名本身就是地址

**编程题答案：**

```c
// 阶乘
int factorial(int n) {
    if(n == 1) return 1;
    return n * factorial(n-1);
}

// 斐波那契
int fib(int n) {
    if(n <= 2) return 1;
    return fib(n-1) + fib(n-2);
}

// 数字各位求和
int digitSum(int n) {
    if(n < 10) return n;
    return n % 10 + digitSum(n / 10);
}

// 汉诺塔
void hanoi(int n, char from, char to, char temp) {
    if(n == 0) return;
    hanoi(n-1, from, temp, to);
    printf("%c -> %c\n", from, to);
    hanoi(n-1, temp, to, from);
}
```
