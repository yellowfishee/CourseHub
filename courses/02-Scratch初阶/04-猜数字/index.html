玩一个游戏：猜数字

老师想一个数字（1~100）

你来猜，老师告诉你：大了 / 小了 / 对了

老师想数字
→
你来猜
→
太大了
或
太小了
或
恭喜！
思考

怎么猜最快？

🎯
每次猜中间的数
100 个数字：先猜 50
大了 → 猜 25 | 小了 → 猜 75
这样能排除一半！

最多 7 次就能猜到！

拆解

游戏需要哪几步？

1
计算机想一个数字（1~100）
2
计算机提问："你要猜多少？"
3
你输入数字
4
计算机比较，告诉你大/小/对
5
继续猜（重复第 2-4 步）
第一步

怎么让计算机想数字？

运算类 → 取随机数 1 到 100

这个积木会产出一个随机数字！

第二步

怎么让计算机提问？

侦测类 → 询问...并等待

🍽️
就像服务员等你点完菜才走
"回答" = 你点的菜（存在这个盒子里）
判断

三种情况

✅
猜对了
说"恭喜你！"
⬆️
猜大了
说"太大了！"
⬇️
猜小了
说"太小了！"

需要用三个「如果」来判断！

常见错误

❌ 错误代码

问题：每次比较时，都会重新产生一个随机数！

就像每次问问题都重新想数字，而不是用之前想的那个

解决方法

用变量记住数字！

📝
变量 = 便利贴
先把随机数存在便利贴上
比较时看便利贴上的数字
取随机数
→
存到"答案"变量
→
每次比较用变量
循环

怎么一直猜下去？

控制类 → 重复执行

重复执行里的积木会一直循环运行！

总结

完整结构

1. 新建变量"答案"，存入随机数
2. 重复执行
3. 询问并等待
4. 如果 回答 > 答案 → 说"太大了"
5. 如果 回答 < 答案 → 说"太小了"
6. 否则 → 说"恭喜！"
