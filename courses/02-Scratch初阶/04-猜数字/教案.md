# 猜数字 - 认识变量、询问和判断

---
title: 猜数字
description: 变量存储、随机数、询问回答、条件判断、循环
prerequisites:  Scratch基础
---

## 学习目标

- 理解**变量**是什么（让计算机记住数据）
- 学会用**随机数**积木生成数字
- 学会用**询问并等待**和**回答**积木
- 理解**条件判断**（如果...那么）
- 学会用**重复执行**循环
- 能够独立完成猜数字游戏

---

## 内容

### 一、引入：玩一个猜数字游戏

**提问**：我们来玩一个游戏，叫"猜数字"。

**规则**：
- 老师想一个数字，范围是 1~100
- 你们来猜
- 老师会告诉你们猜对了、猜大了、还是猜小了

**师生互动**：老师想数字，学生猜（3次以内）

**引导思考**：怎么猜才能最快猜到？

**总结**：每次猜中间的数，这样能排除一半的可能，用最少的次数猜到。

---

### 二、游戏拆解

**提问**：把这个游戏变成计算机程序，需要哪几步？

**拆解问题**：

1. **计算机先想一个数字**（1~100 之间）
2. **计算机提问**："你要猜的数字是多少？"
3. **你输入数字**
4. **计算机比较**：
   - 猜对了 → 结束
   - 猜大了 → 告诉你"大了"
   - 猜小了 → 告诉你"小了"
5. **继续猜**（重复第 2-4 步）

---

### 三、第一步：让计算机想数字

**提问**：怎么让计算机产生一个随机数字？

**引入积木**：运算类里的「取随机数」

![取随机数](assets/image.png)

**解释**：
- 这个积木可以产生一个随机数字
- 范围是 1 到你设置的数字之间

**小任务**：让计算机显示一个 1~100 的随机数

---

### 四、第二步：让计算机提问

**提问**：怎么让计算机问你问题？

**引入积木**：侦测类里的「询问...并等待」

![询问并等待](assets/image-1.png)

**解释**：
- "询问" = 提问
- "并等待" = 等你回答完再继续
- 回答的内容会存在「回答」这个"盒子"里

**生活类比**：

```
询问并等待 = 服务员等你点完菜才走
回答 = 你点的菜
```

**小任务**：让计算机问你"你叫什么名字？"，然后显示你的回答

---

### 五、第三步：比较大小

**提问**：怎么判断你猜的数字和计算机想的数字哪个大？

**引入积木**：控制类的「如果...那么」

**三种情况**：

1. **猜对了** → 说"恭喜你！猜对了！"
2. **猜大了** → 说"太大了！"
3. **猜小了** → 说"太小了！"

**代码示例**：

```
如果 回答 > 电脑想的数字 那么
    说"太大了！"
否则
    如果 回答 < 电脑想的数字 那么
        说"太小了！"
    否则
        说"恭喜你！猜对了！"
```

---

### 六、常见错误：数字没有"记住"

**错误代码**：

![错误代码](assets/image-2.png)

**问题分析**：
- 每次比较时，`取随机数` 都会产生一个新数字！
- 就像每次问问题都重新想一个数字，而不是用之前想的那个

**正确做法**：
- 先把随机数"存"到一个变量里
- 比较时用这个变量

**生活类比**：

```
变量 = 便利贴
先把数字写在便利贴上（存到变量）
每次比较时，看便利贴上的数字（用变量）
```

---

### 七、第四步：重复猜测

**提问**：现在我们只能猜一次，怎么让游戏一直玩下去？

**引入积木**：控制类的「重复执行」

![重复执行](assets/image-3.png)

**解释**：
- 重复执行里的积木会一直循环运行
- 需要配合「询问并等待」才能输入新的答案

**思考任务**：重复执行应该包括哪些积木？

**答案**：
```
重复执行
    询问并等待
    如果...那么...（三种情况的判断）
```

---

### 八、最终挑战

**任务**：完成猜数字游戏，要求：

1. 计算机随机想一个 1~100 的数字
2. 不断提问，直到猜对
3. 每次猜完告诉玩家是大是小
4. 猜对后说"恭喜你！"

**提示**：
- 先把随机数存到变量
- 用重复执行包裹循环
- 用三个如果处理三种情况
