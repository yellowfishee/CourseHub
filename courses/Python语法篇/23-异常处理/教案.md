## 内容

### 一、引入：程序为什么会崩溃？

**问题**：下面的代码有什么问题？

```python
a = int(input("输入数字: "))
print(10 / a)
```

**如果用户输入**：
- "abc" → ValueError（不能转成整数）
- "0" → ZeroDivisionError（不能除以0）

**程序崩溃了！** 用户看到一堆错误信息，体验很差。

**解决方法**：用异常处理，让程序"优雅地"处理错误。

---

### 二、什么是异常？

**异常**：程序运行时发生的错误。

**类比**：
```
异常 = 开车遇到的路障
- 如果没有应对措施 → 撞车（程序崩溃）
- 如果有应对措施 → 绕过去（继续运行）
```

**常见异常类型**：

| 异常 | 含义 | 例子 |
|------|------|------|
| ValueError | 值错误 | int("abc") |
| TypeError | 类型错误 | "a" + 1 |
| ZeroDivisionError | 除以零 | 10 / 0 |
| IndexError | 索引越界 | lst[100] |
| KeyError | 键不存在 | d["not_exist"] |
| FileNotFoundError | 文件不存在 | open("xxx.txt") |

---

### 三、try...except 语句

**基本结构**：

```python
try:
    # 可能出错的代码
except 异常类型:
    # 出错后执行的代码
```

**例子1：处理输入错误**

```python
try:
    a = int(input("输入数字: "))
    print(f"你输入的是: {a}")
except ValueError:
    print("输入无效，请输入数字！")
```

**例子2：处理除以零**

```python
try:
    a = int(input("输入数字: "))
    print(10 / a)
except ValueError:
    print("请输入数字！")
except ZeroDivisionError:
    print("不能除以0！")
```

**例子3：捕获所有异常**

```python
try:
    a = int(input("输入数字: "))
    print(10 / a)
except Exception as e:
    print(f"出错了: {e}")
```

### 四、else 和 finally

**完整结构**：

```python
try:
    # 可能出错的代码
except:
    # 出错时执行
else:
    # 没出错时执行
finally:
    # 无论如何都执行
```

**例子**：

```python
try:
    f = open("data.txt", "r")
    content = f.read()
except FileNotFoundError:
    print("文件不存在")
else:
    print("文件读取成功")
    print(content)
finally:
    print("操作结束")
```

### 五、主动抛出异常

**用 raise 抛出异常**：

```python
def set_age(age):
    if age < 0:
        raise ValueError("年龄不能为负数")
    return age

try:
    set_age(-5)
except ValueError as e:
    print(e)  # 年龄不能为负数
```

### 六、例题：安全输入

**问题**：让用户输入一个正整数，如果输入无效就一直提示

**代码**：

```python
while True:
    try:
        n = int(input("请输入一个正整数: "))
        if n <= 0:
            print("必须是正整数！")
            continue
        break
    except ValueError:
        print("输入无效，请输入数字！")

print(f"你输入的是: {n}")
```

### 七、例题：安全计算器

**问题**：实现一个简单的计算器，处理各种错误

**代码**：

```python
def calculator():
    try:
        a = float(input("第一个数: "))
        op = input("运算符 (+, -, *, /): ")
        b = float(input("第二个数: "))
        
        if op == "+":
            result = a + b
        elif op == "-":
            result = a - b
        elif op == "*":
            result = a * b
        elif op == "/":
            result = a / b
        else:
            print("不支持的运算符")
            return
        
        print(f"结果: {result}")
        
    except ValueError:
        print("请输入有效数字")
    except ZeroDivisionError:
        print("不能除以0")
    except Exception as e:
        print(f"出错了: {e}")

calculator()
```

### 八、常见错误

**错误1：只写except，不写异常类型**

```python
try:
    x = int(input())
except:  # 捕获所有异常，包括KeyboardInterrupt
    print("出错了")
```

**问题**：会捕获所有异常，包括用户按Ctrl+C！

**正确**：明确指定异常类型

```python
try:
    x = int(input())
except ValueError:
    print("请输入数字")
```

**错误2：异常处理太宽泛**

```python
try:
    # 很多代码
except:
    print("出错了")  # 不知道哪里出错
```

**正确**：分开处理

```python
try:
    x = int(input())
except ValueError:
    print("请输入数字")
else:
    try:
        result = 10 / x
    except ZeroDivisionError:
        print("不能除以0")
```

**错误3：不用异常处理而用条件判断**

```python
# 不推荐
x = input()
if x.isdigit():
    x = int(x)
else:
    print("无效")

# 推荐
try:
    x = int(input())
except ValueError:
    print("无效")
```

## 总结

| 概念 | 要点 |
|------|------|
| 异常 | 运行时错误 |
| try | 可能出错的代码 |
| except | 处理异常 |
| else | 没出错时执行 |
| finally | 无论如何都执行 |
| raise | 主动抛出异常 |

**记住**：
- 用 try-except 让程序更健壮
- 明确指定异常类型
- 异常处理不是万能的，有时候让程序崩溃更好
