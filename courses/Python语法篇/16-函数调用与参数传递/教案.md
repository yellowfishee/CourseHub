## 内容

### 一、引入：函数会改变参数吗？

**问题**：下面两个函数，哪个会改变传入的值？

```python
def add_one(n):
    n = n + 1

x = 5
add_one(x)
print(x)  # 输出是 5 还是 6？
```

```python
def add_item(lst):
    lst.append(100)

nums = [1, 2, 3]
add_item(nums)
print(nums)  # 输出是 [1,2,3] 还是 [1,2,3,100]？
```

**答案**：
- 第一个：x 还是 5（没变）
- 第二个：nums 变成 [1, 2, 3, 100]（变了！）

**为什么？** 这涉及到Python的核心概念：**可变对象 vs 不可变对象**

---

### 二、不可变对象

**什么是不可变对象？**

创建后就不能修改的对象。

**不可变类型**：
| 类型 | 例子 | 特点 |
|------|------|------|
| int | 5, 100 | 整数不能改 |
| float | 3.14 | 浮点数不能改 |
| str | "hello" | 字符串不能改 |
| tuple | (1, 2, 3) | 元组不能改 |

**例子**：

```python
x = 5
x = x + 1  # 这不是修改5，而是创建了一个新的6！
```

**类比**：
```
不可变对象 = 刻在石头上的字
你想改？只能重新刻一块新的石头
原来的石头还在那里
```

**传入函数的情况**：

```python
def add_one(n):
    n = n + 1  # 创建了一个新的n，原来的没变

x = 5
add_one(x)
print(x)  # 还是5
```

**解释**：
```
调用前：x → 5
调用时：n → 5（n和x指向同一个5）
函数内：n = n + 1 → n指向新的6
调用后：x还是指向5
```

---

### 三、可变对象

**什么是可变对象？**

创建后可以修改的对象。

**可变类型**：
| 类型 | 例子 | 特点 |
|------|------|------|
| list | [1, 2, 3] | 列表可以改 |
| dict | {"a": 1} | 字典可以改 |
| set | {1, 2, 3} | 集合可以改 |

**例子**：

```python
nums = [1, 2, 3]
nums.append(4)  # 直接修改了列表本身
print(nums)  # [1, 2, 3, 4]
```

**类比**：
```
可变对象 = 白板
你可以随时擦掉重写，还是同一块白板
```

**传入函数的情况**：

```python
def add_item(lst):
    lst.append(100)  # 直接修改了列表本身

nums = [1, 2, 3]
add_item(nums)
print(nums)  # [1, 2, 3, 100] 被改了！
```

**解释**：
```
调用前：nums → [1, 2, 3]
调用时：lst → [1, 2, 3]（lst和nums指向同一个列表）
函数内：lst.append(100) → 列表变成[1, 2, 3, 100]
调用后：nums也指向这个被修改的列表
```

---

### 四、对比总结

| 特性 | 不可变对象 | 可变对象 |
|------|----------|---------|
| 类型 | int, float, str, tuple | list, dict, set |
| 能否修改 | 不能，只能创建新的 | 可以直接修改 |
| 传入函数 | 原来的值不会变 | 可能被修改 |
| 类比 | 刻在石头上 | 写在白板上 |

**关键理解**：
- Python的参数传递是"传递引用"
- 但对于不可变对象，因为不能修改，所以表现得像"值传递"
- 对于可变对象，因为能修改，所以可能被函数改变

---

### 五、如何避免意外修改？

**方法1：传入副本**

```python
def add_item(lst):
    lst.append(100)

nums = [1, 2, 3]
add_item(nums.copy())  # 传入副本
print(nums)  # [1, 2, 3] 原来的没变
```

**方法2：函数内部创建副本**

```python
def safe_add(lst):
    new_lst = lst.copy()  # 创建副本
    new_lst.append(100)
    return new_lst

nums = [1, 2, 3]
result = safe_add(nums)
print(nums)   # [1, 2, 3]
print(result) # [1, 2, 3, 100]
```

### 六、全局变量和局部变量

**什么是作用域？**

变量的"有效范围"。

**局部变量**：在函数内部定义的变量，只在函数内有效

```python
def my_func():
    x = 10  # 局部变量
    print(x)

my_func()
print(x)  # 错误！x在函数外不存在
```

**全局变量**：在函数外部定义的变量，所有地方都能访问

```python
x = 10  # 全局变量

def my_func():
    print(x)  # 可以访问

my_func()  # 10
print(x)    # 10
```

**在函数内修改全局变量**：

```python
x = 10

def change():
    global x  # 声明要用全局变量
    x = 20

change()
print(x)  # 20
```

> ⚠️ **注意**：不写 `global` 的话，会在函数内创建一个同名局部变量，不会修改全局变量！

---

### 七、常见错误

**错误1：以为整数会被函数修改**

```python
def add_one(n):
    n = n + 1

x = 5
add_one(x)
print(x)  # 还是5！
```

**正确做法**：用返回值

```python
def add_one(n):
    return n + 1

x = 5
x = add_one(x)  # 用返回值更新
print(x)  # 6
```

**错误2：意外修改了传入的列表**

```python
def process(lst):
    lst.append(0)  # 意外修改

nums = [1, 2, 3]
process(nums)
print(nums)  # [1, 2, 3, 0] 被改了！
```

**正确做法**：传入副本或函数内创建副本

---

## 总结

| 概念 | 要点 |
|------|------|
| 不可变对象 | int, float, str, tuple，创建后不能改 |
| 可变对象 | list, dict, set，可以修改 |
| 参数传递 | 传递引用，但不可变对象不会被修改 |
| 局部变量 | 只在函数内有效 |
| 全局变量 | 用 `global` 声明才能修改 |

**记住**：
- 整数、字符串传入函数不会被修改
- 列表、字典传入函数可能被修改
- 用 `.copy()` 可以避免意外修改
