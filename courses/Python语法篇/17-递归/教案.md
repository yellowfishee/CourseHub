## 内容

### 一、引入：什么是递归？

**生活例子**：
```
故事：《俄罗斯套娃》
大娃娃里有个小娃娃
小娃娃里还有个更小的娃娃
... 直到最小的娃娃没有娃娃了

这就像递归：打开一个娃娃，发现里面还是娃娃（调用自己）
直到发现一个没有娃娃的（基准情形）
```

**定义**：
```
递归 = 函数调用自己
```

**类比**：
```
递归 = 打开套娃
每次打开都发现里面还有一个娃娃
直到发现最后一个没有娃娃的，就开始往外装回去
```

### 二、递归的两个条件

**条件1：基准情形（Base Case）**

必须有"停止点"，否则会无限调用！

**条件2：递归调用（Recursive Call）**

每次调用都要"缩小问题"，向基准情形靠近。

**例子**：计算1到n的和

```python
def sum_to_n(n):
    # 基准情形：n=1时，直接返回1
    if n == 1:
        return 1
    
    # 递归调用：n + 前面n-1的和
    return n + sum_to_n(n - 1)

print(sum_to_n(5))  # 15
```

**执行过程**：
```
sum_to_n(5) → 5 + sum_to_n(4)
             → 5 + 4 + sum_to_n(3)
             → 5 + 4 + 3 + sum_to_n(2)
             → 5 + 4 + 3 + 2 + sum_to_n(1)
             → 5 + 4 + 3 + 2 + 1  ← 基准情形
             → 15
```

### 三、例题：阶乘的递归写法

**阶乘定义**：
```
n! = n × (n-1) × (n-2) × ... × 1
```

**递归思路**：
```
n! = n × (n-1)!
    = n × (n-1) × (n-2)!
    = ... 直到 1! = 1
```

**代码**：

```python
def factorial(n):
    # 基准情形
    if n == 1:
        return 1
    
    # 递归调用
    return n * factorial(n - 1)

print(factorial(5))   # 120
print(factorial(10))  # 3628800
```

**执行过程可视化**：
```
factorial(5)
├── 5 * factorial(4)
│   ├── 4 * factorial(3)
│   │   ├── 3 * factorial(2)
│   │   │   ├── 2 * factorial(1)
│   │   │   │   └── return 1  ← 基准情形
│   │   │   └── return 2*1 = 2
│   │   └── return 3*2 = 6
│   └── return 4*6 = 24
└── return 5*24 = 120
```

### 四、例题：斐波那契数列

**斐波那契数列**：
```
1, 1, 2, 3, 5, 8, 13, 21, ...

规律：第n项 = 第n-1项 + 第n-2项
```

**递归思路**：
```python
fib(n) = fib(n-1) + fib(n-2)
```

**代码**：

```python
def fib(n):
    # 基准情形：第1和第2项都是1
    if n == 1 or n == 2:
        return 1
    
    # 递归调用
    return fib(n - 1) + fib(n - 2)

print(fib(1))  # 1
print(fib(5))  # 5
print(fib(10)) # 55
```

**执行过程可视化**：
```
fib(5)
├── fib(4)
│   ├── fib(3)
│   │   ├── fib(2) = 1
│   │   └── fib(1) = 1
│   │   └── return 1+1 = 2
│   └── fib(2) = 1
│   └── return 2+1 = 3
└── fib(3)
    ├── fib(2) = 1
    └── fib(1) = 1
    └── return 1+1 = 2
└── return 3+2 = 5
```

### 五、递归的深度限制

**问题**：Python对递归深度有限制！

```python
def countdown(n):
    if n == 0:
        return
    countdown(n - 1)

countdown(1000)   # 正常
countdown(2000)   # 可能报错！
```

**错误信息**：
```
RecursionError: maximum recursion depth exceeded
```

**原因**：Python默认只允许约1000层递归调用。

**解决方法**：

```python
import sys
sys.setrecursionlimit(10000)  # 设置为10000层

def countdown(n):
    if n == 0:
        return
    countdown(n - 1)

countdown(5000)  # 现在可以了
```

> ⚠️ **注意**：递归深度太大可能导致内存溢出，建议用循环代替！

### 六、递归 vs 循环

**同一个问题，两种写法**：

| 问题 | 递归写法 | 循环写法 |
|------|---------|---------|
| 阶乘 | `n * factorial(n-1)` | `for i in range(1, n+1)` |
| 求和 | `n + sum_to_n(n-1)` | `for i in range(1, n+1)` |

**对比**：

| 特点 | 递归 | 循环 |
|------|------|------|
| 代码简洁度 | 通常更简洁 | 可能更长 |
| 性能 | 较慢（函数调用开销） | 较快 |
| 内存占用 | 较大（调用栈） | 较小 |
| 可读性 | 对某些问题更直观 | 对某些问题更直观 |

**何时用递归？**
- 问题本身是递归定义的（如斐波那契）
- 需要处理嵌套结构（如树、图）
- 代码简洁性更重要

**何时用循环？**
- 需要高性能
- 递归深度可能很大
- 简单的重复操作

### 七、常见错误

**错误1：没有基准情形**

```python
def bad_recursion(n):
    return n + bad_recursion(n - 1)  # 永远不会停止！

bad_recursion(5)  # RecursionError!
```

**错误2：基准情形写错**

```python
def factorial(n):
    if n == 0:  # 错！应该是 n == 1
        return 0
    return n * factorial(n - 1)

factorial(5)  # 结果是0，因为最后乘了0
```

**错误3：递归深度太大**

```python
def fib(n):
    if n == 1 or n == 2:
        return 1
    return fib(n - 1) + fib(n - 2)

fib(50)  # 非常慢！可能超时
```

> ⚠️ **斐波那契的递归写法很慢**，因为会重复计算很多次！

## 总结

| 概念 | 要点 |
|------|------|
| 递归 | 函数调用自己 |
| 基准情形 | 停止条件，必须有！ |
| 递归调用 | 每次缩小问题规模 |
| 深度限制 | 默认约1000层，可设置 |

**记住**：
- 递归必须有基准情形
- 每次调用都要向基准情形靠近
- 深度太大用 `sys.setrecursionlimit()`
- 性能重要时考虑用循环代替
