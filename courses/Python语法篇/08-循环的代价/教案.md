## 内容

### 一、引入：循环要花多少时间？

**问题**：下面两个代码哪个更快？

```python
# 代码A：循环100次
for i in range(100):
    print(i)
```

```python
# 代码B：循环1000000次
for i in range(1000000):
    print(i)
```

**答案**：代码A更快，因为循环次数少！

**类比**：
```
循环就像跑步：
跑100圈 = 快
跑1000000圈 = 慢

循环次数越多，花的时间越长
```

---

### 二、如何计算循环次数？

**单层循环**：

```python
for i in range(100):  # 循环100次
    print(i)
```

**双层循环（嵌套）**：

```python
for i in range(10):       # 外层循环10次
    for j in range(10):   # 内层循环10次
        print(i, j)
```

**问题**：上面代码总共循环多少次？

**答案**：10 × 10 = 100 次！

**规律**：
```
嵌套循环的总次数 = 外层次数 × 内层次数
```

---

### 三、时间复杂度的概念

**什么是时间复杂度？**

描述代码执行次数的增长趋势。

| 循环次数 | 表示法 | 名称 |
|---------|--------|------|
| N 次 | O(N) | 线性 |
| N² 次 | O(N²) | 平方 |
| N³ 次 | O(N³) | 立方 |
| 常数次 | O(1) | 常数 |

**类比**：
```
O(N) = 跑 N 圈
O(N²) = 跑 N×N 圈
O(N³) = 跑 N×N×N 圈

圈数越多，越累！
```

**对比表**：

| N 的值 | O(1) | O(N) | O(N²) | O(N³) |
|-------|------|------|-------|-------|
| 10 | 1 | 10 | 100 | 1000 |
| 100 | 1 | 100 | 10000 | 1000000 |
| 1000 | 1 | 1000 | 1000000 | 很大！ |

> ⚠️ **注意**：N² 和 N³ 增长得非常快！1000³ = 1,000,000,000（十亿次）！

---

### 四、实例分析

**例题1：O(N) 的代码**

```python
n = 1000
for i in range(n):  # 循环1000次
    print(i)
```

执行次数：1000 次 = O(N)

---

**例题2：O(N²) 的代码**

```python
n = 1000
for i in range(n):      # 外层1000次
    for j in range(n):  # 内层1000次
        print(i, j)
```

执行次数：1000 × 1000 = 1,000,000 次 = O(N²)

---

**例题3：O(N²) 变体**

```python
n = 1000
for i in range(n):
    for j in range(i):  # 内层循环次数不固定！
        print(i, j)
```

**问题**：总共循环多少次？

**分析**：
```
i=0: 内层循环 0 次
i=1: 内层循环 1 次
i=2: 内层循环 2 次
...
i=999: 内层循环 999 次

总次数 = 0+1+2+...+999 = 999×1000/2 ≈ N²/2
```

结论：仍然是 O(N²)，只是系数不同。

### 五、竞赛中的时间限制

**竞赛规则**：

| 时间限制 | 能承受的复杂度 |
|---------|---------------|
| 1秒 | O(N) ≈ 10⁷ |
| 1秒 | O(N²) ≈ 10³~10⁴ |
| 1秒 | O(N³) ≈ 10² |

**判断方法**：
```
如果 N = 100000：
  O(N) → 可以
  O(N²) → 太慢！会超时
  O(N³) → 更慢！肯定超时
```

**例题**：N=10000 时，O(N²) 会执行多少次？

答案：10000 × 10000 = 100,000,000 次（一亿次）
大约需要几秒，可能超时！

### 六、如何优化循环？

**优化思路1**：减少循环层数

```python
# 原代码：O(N²)
for i in range(n):
    for j in range(n):
        if i == j:
            print(i)
```

```python
# 优化：O(N)
for i in range(n):
    print(i)  # 只需要一层循环
```

---

**优化思路2**：减少不必要的操作

```python
# 原代码：每次都计算
for i in range(n):
    result = i * (i + 1) / 2
    print(result)
```

```python
# 优化：提前计算不变的值
temp = 0.5  # 只计算一次
for i in range(n):
    result = i * (i + 1) * temp
    print(result)
```

---

**优化思路3**：利用数学公式

```python
# 原代码：O(N) 求和
total = 0
for i in range(1, n+1):
    total += i
```

```python
# 优化：O(1) 用公式
total = n * (n + 1) // 2
```

**类比**：
```
方法1：一个一个数（循环）
方法2：直接用公式计算（数学）
公式更快！
```

### 七、嵌套循环的代价

**问题**：三层嵌套循环呢？

```python
n = 100
for i in range(n):
    for j in range(n):
        for k in range(n):
            print(i, j, k)
```

执行次数：100 × 100 × 100 = 1,000,000 次 = O(N³)

**类比**：
```
一层循环 = 跑圈
二层循环 = 每圈跑N次小圈
三层循环 = 每小圈再跑N次更小的圈

累死了！
```

---

### 八、判断代码是否会超时

**步骤**：

1. 看 N 的值（题目会给出）
2. 计算代码的时间复杂度
3. 对照时间限制表格

**例题**：N=10000，时间限制1秒，下面代码能通过吗？

```python
n = 10000
for i in range(n):
    for j in range(n):
        print(i * j)
```

**分析**：
- 时间复杂度：O(N²)
- 执行次数：10000 × 10000 = 100,000,000 次
- 结论：可能超时！需要优化

---

### 九、常见错误

**错误**：以为 O(N²) 永远可以

```python
# N=100000 时，这个代码会超时！
n = 100000
for i in range(n):
    for j in range(n):
        if i * j == 12345:
            print(i, j)
```

**正确做法**：先判断 N 的大小，再选择算法

---

## 总结

| 概念 | 含义 |
|------|------|
| 时间复杂度 | 执行次数的增长趋势 |
| O(N) | 单层循环 |
| O(N²) | 双层嵌套循环 |
| O(N³) | 三层嵌套循环 |
| 超时 | 执行次数太多，超过时间限制 |

**记住**：
- 循环层数越多，代价越大
- N 大时，O(N²) 可能超时
- 学会用公式代替循环
