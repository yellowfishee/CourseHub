## 内容

### 一、递归应用场景

**适合递归的问题**：
- 问题可以分解成更小的同类问题
- 有明确的终止条件
- 不需要很大的递归深度

**例子**：
- 阶乘计算
- 斐波那契数列
- 汉诺塔问题
- 二叉树遍历

---

### 二、例题1：阶乘的多种写法

**递归写法**：

```python
def factorial_recursive(n):
    if n == 1:
        return 1
    return n * factorial_recursive(n - 1)

print(factorial_recursive(5))  # 120
```

**迭代写法**：

```python
def factorial_iterative(n):
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result

print(factorial_iterative(5))  # 120
```

**对比**：
| 特点 | 递归 | 迭代 |
|------|------|------|
| 代码长度 | 短 | 较长 |
| 可读性 | 好（数学定义直观） | 一般 |
| 效率 | 低（函数调用开销） | 高 |
| 内存 | 多（调用栈） | 少 |

### 三、例题2：斐波那契数列

**问题**：计算斐波那契数列的第n项

**递归写法（不推荐）**：

```python
def fib_recursive(n):
    if n == 1 or n == 2:
        return 1
    return fib_recursive(n - 1) + fib_recursive(n - 2)

# 问题：效率极低！
# fib_recursive(50) 可能要几分钟
```

**迭代写法（推荐）**：

```python
def fib_iterative(n):
    if n == 1 or n == 2:
        return 1
    a, b = 1, 1
    for i in range(3, n + 1):
        a, b = b, a + b
    return b

print(fib_iterative(50))  # 快速得到结果
```

**效率对比**：
```
n = 10: 递归 0.001秒 vs 迭代 0.000秒
n = 30: 递归 0.3秒 vs 迭代 0.000秒
n = 50: 递归 数分钟 vs 迭代 0.000秒
```

### 四、例题3：汉诺塔

**问题**：把n个盘子从A柱移动到C柱，借助B柱

**规则**：
- 每次只能移动一个盘子
- 大盘子不能放在小盘子上面

**递归思路**：
```
1. 把n-1个盘子从A移到B（借助C）
2. 把第n个盘子从A移到C
3. 把n-1个盘子从B移到C（借助A）
```

**代码**：

```python
def hanoi(n, from_pole, to_pole, aux_pole):
    if n == 1:
        print(f"从{from_pole}移到{to_pole}")
        return
    
    hanoi(n - 1, from_pole, aux_pole, to_pole)
    print(f"从{from_pole}移到{to_pole}")
    hanoi(n - 1, aux_pole, to_pole, from_pole)

hanoi(3, 'A', 'C', 'B')
```

**输出**：
```
从A移到C
从A移到B
从C移到B
从A移到C
从B移到A
从B移到C
从A移到C
```

### 五、例题4：求列表的最大值（递归）

```python
def max_recursive(lst):
    if len(lst) == 1:
        return lst[0]
    
    sub_max = max_recursive(lst[1:])
    return lst[0] if lst[0] > sub_max else sub_max

print(max_recursive([3, 5, 1, 8, 2]))  # 8
```

### 六、例题5：判断回文（递归）

```python
def is_palindrome(s):
    if len(s) <= 1:
        return True
    
    if s[0] != s[-1]:
        return False
    
    return is_palindrome(s[1:-1])

print(is_palindrome("abcba"))  # True
print(is_palindrome("hello"))  # False
```

### 七、何时用递归？

**推荐用递归**：
- 问题本身就是递归定义的
- 代码简洁性很重要
- 递归深度不会太大

**不推荐用递归**：
- 性能很重要
- 递归深度可能很大（如斐波那契）
- 迭代写法也很简单

---

## 总结

| 概念 | 要点 |
|------|------|
| 递归优点 | 代码简洁，符合数学定义 |
| 递归缺点 | 效率低，内存占用大 |
| 迭代优点 | 效率高，内存占用少 |
| 迭代缺点 | 代码可能较长 |

**记住**：
- 斐波那契用迭代，不用递归
- 汉诺塔适合用递归
- 性能重要时，优先用迭代
